# Error Handling on REST API

Every error must become a proper HTTP response.

## Standard Error Format

Always return structured errors:

HTTP 400
{
    "error": {
        "code": "VALIDATION_ERROR",
        "message": "Email is not valid",
        "details": {"field": "email", "reason": "format"},
        "request_id": "req_abc123"
    }
}

Codes are machine-readable. Messages are human-readable.
request_id is for debugging.

## Status Codes

Success:
200 - GET, PATCH
201 - POST
204 - DELETE (no body)

Client Errors:
400 - Bad request (validation)
401 - Unauthenticated (no token)
403 - Forbidden (wrong role)
404 - Not found
409 - Conflict (duplicate)
422 - Validation error (Unprocessable)
429 - Too many requests (rate limit)

Server Errors:
500 - Internal error (unhandled exception)
502 - Bad gateway (upstream failed)
503 - Service unavailable (overloaded)
504 - Gateway timeout

## Global Error Handler

Instead of try/except in every handler:

from fastapi import Request
from fastapi.responses import JSONResponse

@app.exception_handler(AppError)
async def app_error_handler(request: Request, exc: AppError):
    status_codes = {
        "NOT_FOUND": 404,
        "VALIDATION": 400,
        "UNAUTHORIZED": 401,
        "FORBIDDEN": 403,
        "CONFLICT": 409,
    }
    return JSONResponse(
        status_code=status_codes.get(exc.code, 500),
        content={
            "error": {
                "code": exc.code,
                "message": str(exc),
                "request_id": request.state.request_id,
            }
        },
    )

@app.exception_handler(Exception)
async def unexpected_error_handler(request: Request, exc: Exception):
    logger.critical("Unhandled exception", exc_info=True)
    return JSONResponse(
        status_code=500,
        content={
            "error": {
                "code": "INTERNAL_ERROR",
                "message": "Internal server error",
                "request_id": request.state.request_id,
            }
        },
    )

## Categories of Errors

1. Validation errors (400) - Client sent bad data
2. Auth errors (401/403) - Client not allowed
3. Not found (404) - Resource missing
4. Conflict (409) - State conflict
5. External errors (502/503) - Upstream service failed
6. Internal errors (500) - Bugs, unhandled cases

Each category has its own handling strategy.
