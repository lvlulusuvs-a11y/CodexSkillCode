# Logging Best Practices

Good logging is the difference between debugging in 5 minutes
and debugging for 5 hours.

## Structured Logging

Log in JSON format, not plain text.

BAD:
[2026-05-31 12:00:00] INFO: Order created for user 123

GOOD:
{"time":"2026-05-31T12:00:00Z","level":"INFO","message":"order_created","order_id":"ord_123","user_id":"usr_123","total":49.99,"duration_ms":150}

JSON logs can be parsed, searched, and graphed.

## Using structlog

import structlog

structlog.configure(
    processors=[
        structlog.stdlib.filter_by_level,
        structlog.stdlib.add_log_level,
        structlog.processors.TimeStamper(fmt="iso"),
        structlog.processors.JSONRenderer(),
    ],
)

logger = structlog.get_logger()

## Contextual Logging

Add context to every log:

logger.info("user_registered", user_id="u123", source="web")

## What to Log

Log:
- Every request (method, path, status, duration)
- Every external call (service, duration, result)
- Every business event (order_created, payment_processed)
- Every error (code, context, traceback)

Never Log:
- Passwords (anywhere)
- Tokens and API keys
- Personal data (emails outside of structured fields)
- Credit card numbers
- Full request bodies

## Log Levels

DEBUG: Development details (off in production)
INFO: Business events and state changes
WARN: Anomalies that dont break anything
ERROR: Errors affecting one request
CRITICAL: System-level failures

## Correlation ID

Every request gets a unique ID, propagated through the system.

request = {
    "request_id": "req_abc123"
}

forward_to_service(request_id=request_id)
