# Remote Work for Engineers

Remote work is a skill. Here is how to do it well.

## Communication

Over-communicate:

1. Write async-first (Slack, docs), not sync-first (meetings)
2. Share context liberally (what, why, status)
3. Record decisions (written, not verbal)
4. Default to transparency (public channels, not DMs)

## Daily Rhythm

1. Start day with standup update
2. Deep work blocks (2-3 hours, no interruptions)
3. Check messages between deep work blocks
4. End day with summary of what was done
5. Clear communication about availability

## Writing Culture

Write well because everything is written:

1. Design docs are written and reviewed async
2. Decisions are documented
3. Knowledge is shared (not kept in heads)
4. Questions are answered in public channels

## Avoiding Burnout

1. Separate work and life (physical space if possible)
2. Stop at the same time each day
3. Take breaks (walk, stretch, eat away from desk)
4. Socialize with colleagues (virtual coffee)
5. Exercise
6. Have hobbies outside of work

## Managing Up

1. Give status updates proactively (dont wait to be asked)
2. Flag problems early (with proposed solutions)
3. Communicate progress (what you shipped, not what you did)
4. Set boundaries (availability hours)

## Tools

1. Slack or Discord for chat
2. Zoom or Google Meet for video
3. GitHub or GitLab for code
4. Notion or Confluence for docs
5. JIRA or Linear for tickets
6. Miro or Figma for diagrams

## Common Mistakes

1. Not communicating enough
2. Working too much (no boundaries)
3. Not socializing (isolation)
4. Skipping video calls (losing human connection)
5. Not investing in relationships
