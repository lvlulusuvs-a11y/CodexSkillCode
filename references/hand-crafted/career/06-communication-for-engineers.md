# Communication for Engineers

Technical skills get you hired. Communication skills get you promoted.

## Writing

### Technical Design Docs

1. One-page summary first
2. Context before details
3. Options with trade-offs, not recommendations
4. Diagrams are worth 1000 words
5. Review before writing code

### Status Updates

Good: "I migrated the payment service. 100% test pass rate. Cutover planned for Thursday."
Bad: "Working on it."

### Code Review Comments

Good: "This is a hot path (100 rps). The current implementation does N+1 queries.
Suggested fix: use eager loading. I can pair if needed."
Bad: "Fix N+1."

## Speaking

### Presenting to Engineers

1. Start with the problem, not the solution
2. Show data, not opinions
3. Be concise (respect their time)
4. Leave time for questions

### Presenting to Non-Engineers

1. No jargon
2. Focus on business value
3. Use analogies
4. Be clear about risks and timelines

## Meetings

### Before

- Do I need to be here?
- What is the outcome I want?
- What preparation is needed?

### During

- Listen more than you speak
- Take notes
- Clarify action items
- Stay on topic

### After

- Send notes within 24 hours
- Follow up on action items
- Cancel if not needed

## Difficult Conversations

1. State the facts (not interpretations)
2. Explain the impact (why it matters)
3. Ask for their perspective
4. Propose a solution together
5. Agree on next steps
