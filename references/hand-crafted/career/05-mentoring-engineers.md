# Mentoring Engineers: Leveling Up Your Team

Mentoring is the highest-leverage activity for a senior engineer.

## The Mentoring Mindset

1. Their success is your success
2. Teaching forces you to understand deeply
3. Good mentors create more mentors
4. The best engineers are teachers

## How to Mentor Effectively

### 1. Pair Programming

Best way to transfer knowledge:
- Drive/navigate: one types, one reviews
- Explain decisions as you make them
- Let them drive (even if slower)
- Focus on process, not just code

### 2. Code Review as Mentoring

Write reviews that teach:

BAD: "Fix this"
GOOD: "This variable leaks memory because it captures the closure.
Try using WeakRef or moving it outside the loop.
Here is an example: ..."

### 3. Design Reviews

Have junior engineers write design docs:
- Review their design, not just their code
- Ask questions that guide, not dictate
- Let them make mistakes in design (cheaper than in code)
- Praise good thinking, even when wrong

### 4. Growing Ownership

Gradually transfer ownership:
Week 1: Watch you do it
Week 2: Do it together
Week 3: They do it, you review
Week 4: They own it, you mentor others

## Common Mentoring Mistakes

1. Doing it for them (they learn nothing)
2. Overwhelming them (too much information)
3. Only giving negative feedback
4. Not setting clear expectations
5. Comparing them to yourself

## Building a Learning Culture

1. Weekly tech talks
2. Book club
3. Coding dojos
4. Conference attendance
5. Internal workshops
6. Open source contributions
7. Bug bashes
