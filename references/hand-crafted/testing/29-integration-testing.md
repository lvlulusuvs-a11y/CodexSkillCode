# Integration Testing Strategies

Integration tests catch bugs that unit tests miss.

## What to Test

Service interactions:
- API -> Database
- API -> Cache
- API -> External service (mocked)
- Queue consumer -> Database
- Webhook -> Database

## Test Database

Use a real database, not a mock.

import pytest
from testcontainers.postgres import PostgresContainer

@pytest.fixture(scope="session")
def postgres():
    with PostgresContainer("postgres:16") as pg:
        yield pg.get_connection_url()

@pytest.fixture
async def db_pool(postgres):
    pool = await asyncpg.create_pool(postgres)
    yield pool
    await pool.close()

## Test Containers

TestContainers creates real instances of databases, queues, caches.

from testcontainers.redis import RedisContainer
from testcontainers.kafka import KafkaContainer

with RedisContainer("redis:7") as redis:
    client = redis.get_client()
    client.set("key", "value")
    assert client.get("key") == b"value"

## Transaction Rollback

For tests that modify the database, roll back changes:

@pytest.fixture
async def transaction(db_pool):
    async with db_pool.acquire() as conn:
        async with conn.transaction():
            yield conn
            raise  # Rollback!

## Mocking External HTTP APIs

Use responses or respx to mock HTTP calls.

import respx
from httpx import Response

@respx.mock
async def test_external_api():
    respx.get("https://api.example.com/data").mock(
        return_value=Response(200, json={"key": "value"})
    )

    result = await my_service.fetch_data()
    assert result["key"] == "value"

## Integration Test Checklist

- [ ] Test with real database, not SQLite in memory
- [ ] Test with real cache, not a dict mock
- [ ] Test database connection failures
- [ ] Test cache connection failures
- [ ] Test timeouts
- [ ] Test concurrent access
- [ ] Test data integrity after restart
